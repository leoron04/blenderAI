# Linee guida per le GitHub Discussions

Questa guida definisce come usare le Discussions di BlenderAI in modo ordinato e utile per la community.

## Categorie disponibili
- **General**: annunci non bloccanti, feedback generico, roadmap condivisa.
- **Q&A**: richieste di supporto, how-to, troubleshooting.
- **Announcements**: comunicazioni ufficiali del team (rilasci, note importanti). Solo i maintainer dovrebbero creare thread qui.

## Regole di buona condotta
1. Cercare prima di postare: usa la ricerca per evitare duplicati.
2. Sii conciso: un thread = un argomento, titolo descrittivo.
3. Condividi contesto minimo: versione Blender, versione addon, OS, log/traceback e screenshot se rilevanti.
4. Segnala lo stato: usa le template dedicate (vedi sotto) per rendere l’indagine più rapida.
5. Moderazione: spam, linguaggio offensivo o contenuti fuori tema saranno chiusi.

## Template disponibili
- **Feature Request** (`General`): idee e nuove funzionalità.
- **Bug Report** (`General`): problemi riproducibili che non richiedono un fix immediato in Issues.
- **How-To Question** (`Q&A`): richieste di supporto operativo o clarifiche d’uso.

## Thread di benvenuto (pinnato)
- Pubblica e mantieni pinnato il post “Welcome to BlenderAI - Intelligent AI Assistant for Blender!” nella categoria **Announcements**. Il contenuto proposto è in `docs/discussions/welcome-to-blenderai.md`.

## Quando aprire una Issue invece di una Discussion
- Bug con impatto diretto sulla produzione o blocchi di installazione ➜ Issue (usa il template Bug Report).
- Vulnerabilità o problemi di sicurezza ➜ Issue privata / canale sicuro.
- Richieste di feature già validate ➜ Issue con specifiche chiare.

## Moderazione e triage
- Sposta i thread nella categoria corretta se necessario.
- Aggiorna il titolo per riflettere l’argomento.
- Marca le soluzioni utili con “Mark as answer” nelle Q&A per migliorare la ricerca.
