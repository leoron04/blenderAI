# Contributing to BlenderAI

Thank you for your interest in contributing to BlenderAI! Here's how you can help:

## Getting Started

1. Fork the repository
2. Clone your fork: `git clone https://github.com/YOUR-USERNAME/blenderAI.git`
3. Create a branch: `git checkout -b feature/your-feature-name`
4. Install dependencies: `pip install -r requirements.txt`
5. Make your changes
6. Test your code
7. Commit: `git commit -m "Add your feature"`
8. Push: `git push origin feature/your-feature-name`
9. Open a Pull Request

## Development Guidelines

### Code Style
- Follow PEP 8 standards
- Use meaningful variable and function names
- Add docstrings to all functions
- Write type hints where possible

### Testing
- Test your code thoroughly
- Include comments explaining complex logic
- Ensure no breaking changes

### Commit Messages
- Use clear, descriptive commit messages
- Start with a verb (Add, Fix, Update, etc.)
- Reference issues when relevant: "Fix #123"

## Areas for Contribution

- **Bug Fixes**: Report and fix issues
- **Features**: Implement new AI capabilities
- **Documentation**: Improve README and docs
- **Testing**: Add unit tests
- **Performance**: Optimize code
- **Translations**: Add language support

## Reporting Bugs

Include:
- Description of the bug
- Steps to reproduce
- Expected behavior
- Actual behavior
- Blender version
- Addon version

## Feature Requests

Describe:
- The feature you want
- Why it's useful
- Potential implementation approach

## Code Review Process

1. Maintainers will review your PR
2. You may be asked to make changes
3. Once approved, your code will be merged
4. Your contribution will be credited

## Questions?

Open an issue or discussion thread. We're here to help!

Thank you for contributing! 🙏
